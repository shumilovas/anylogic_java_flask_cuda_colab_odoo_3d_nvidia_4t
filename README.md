# anylogic_java_flask_cuda_colab_odoo_3d_nvidia_4t

### Цифровой Двойник Склада: Сквозная интеграция имитационного моделирования (AnyLogic), ИИ-вычислений на GPU (NVIDIA CUDA) и промышленного WMS/ERP-контура (Odoo 17)

Реализация высоконагруженного отказоустойчивого MVP-конвейера Smart Factory в рамках концепции **Industry 4.0**. Система в реальном времени связывает симуляцию физических процессов, распределенный анализ геометрии объектов на видеокартах NVIDIA и автоматизацию складской логистики предприятия.

---

## 🏗️ Архитектура системы и потоки данных

Взаимодействие компонентов спроектировано по событийно-ориентированному принципу (**Event-Driven**) для минимизации оверхеда на сеть:





[ AnyLogic (Java Симуляция) ]│▼ (POST-запросы / Координаты и Брак)[ Flask API Gateway (Python) ] ──(Туннель loca.lt)──► [ NVIDIA CUDA / CuPy (Google Colab) ]│                                                (Анализ физики PhysX Core, 50 FPS)▼ (Протокол XML-RPC / Порт 8069)[ Odoo 17 ERP + СУБД PostgreSQL 15 ] (Docker Стек)│▼[ Инвентаризация остатков: SKU E-COM12 ]



# 🚀 Ключевой функционал компонентов

*   **Имитационное моделирование (AnyLogic 8.8)**: Динамическая симуляция конвейерной линии. Логика отправки данных вынесена на Java-триггеры блоков `On Enter`. Это исключает избыточный polling-опрос сети и снижает нагрузку на бэкенд.
*   **Ускорение вычислений (NVIDIA CUDA / CuPy)**: Матричный real-time анализ векторов координат, коллизий и геометрии объектов симуляции на базе облачного GPU ядра (NVIDIA Tesla T4). 
*   **Отказоустойчивый шлюз (Flask API)**: Изолирующая API-прослойка бэкенда [1.1]. Реализован безопасный контекст транзакций `try-except` [1.1]. Любые сетевые тайм-ауты или строгие блокировки со стороны ERP локализуются внутри шлюза [1.1]. Система гарантированно возвращает `status: 200`, защищая 3D-стрим фронтенда (WebGL/Three.js) и AnyLogic от падений (`ERR_CONNECTION_RESET`) [1.1].
*   **Учет и автоматизация (Odoo 17 ERP)**: При обнаружении дефекта (`status: damaged`) Flask-сервер инициирует удаленный вызов процедур к Odoo [1.1]. Система находит карточку товара по внутреннему артикулу (`Internal Reference`: `E-COM12`) и автоматически корректирует или инициализирует складские кванты (`stock.quant`) на локации `WH/Stock` [1.3].

## 🛠️ Инструкция по быстрому запуску

### Шаг 1. Инициализация инфраструктуры Odoo WMS
Разверните изолированный стек ERP-системы и СУБД PostgreSQL 15 в пространстве имен проекта:
```bash
docker-compose -p wms_project up -d
```

### Шаг 2. Развертывание виртуального окружения Python
Установите необходимые системные библиотеки бэкенда [1.1]:
```bash
pip install -r requirements.txt
```
*(Содержимое `requirements.txt`: `flask`, `flask-cors`, `requests`, `gunicorn`)* [1.1].

### Шаг 3. Настройка ИИ-конвейера и запуск шлюза
1. Запустите исполняемую ячейку CuPy-анализа в Google Colab и скопируйте сгенерированный публичный URL (`https://loca.lt`) [1.1].
2. Пропишите полученный адрес в `wms_stats_server.py` в переменную `COLAB_CUDA_URL` [1.1].
3. Запустите сервер:
```bash
python wms_stats_server.py
```
4. Включите модель в AnyLogic.
